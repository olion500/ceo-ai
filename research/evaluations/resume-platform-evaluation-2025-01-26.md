# Business Idea Evaluation: 경력자 이력서 정리 플랫폼

## Executive Summary

**One-line pitch:** Slack/Jira 연동으로 경력자의 업무 성과를 자동 추출하고 채용공고 맞춤형 이력서를 생성하며 크라우드소싱 면접 질문으로 면접을 준비시켜주는 플랫폼

**Target market:** 이직 준비 중인 한국 경력 개발자/PM (3년차 이상)

**Business model:** 월 구독제 (프리미엄 기능)

**Unique angle:** Slack/Jira 자동 연동으로 "내가 뭘 했는지" 자동 정리 (vs Poromy의 수동 입력 프롬프트 생성)

## Detailed Evaluation

### 1. Problem Intensity: 7/10

**Evidence:**
- 경력자들이 이직 시 "내가 뭘 했는지" 정리하는 것이 실제로 고통스러운 작업
- 여러 프로젝트를 거친 경력자일수록 기억과 정리가 어려움
- 채용공고마다 강조점이 달라 맞춤형 이력서 작성 필요
- 면접 질문 준비는 불안감이 큰 영역

**Reasoning:**
실제 pain point이지만 "긴급한" 문제는 아님. 이직 준비 기간(보통 1-3개월)이 있어서 시간을 들여 해결 가능. 하지만 시간을 절약하고 퀄리티를 높일 수 있다면 기꺼이 돈을 낼 의향이 있는 타겟. 특히 고연봉 경력자일수록 시간 대비 가치가 높음.

**Confidence Level:** High

**Key Assumptions:**
- 경력자들이 Slack/Jira를 실제 업무 도구로 사용 중 (스타트업/중견기업)
- 이력서 작성을 "귀찮고 어려운 일"로 인식

### 2. Market Size: 6/10

**Evidence:**
- TAM (Total Addressable Market): 한국 IT 경력자 약 50만명
- SAM (Serviceable Available Market): 연간 이직 준비자 약 5-10만명 (10-20%)
- SOM (Serviceable Obtainable Market, Year 1): 500-1,000명 (유료 전환 1-2%)

**Reasoning:**
국내 시장에 한정되어 있고 (한국 이력서 문화 특화), "이력서 작성 도구에 돈을 낼" 사람은 전체 이직자 중 일부. 하지만 경력자 타겟이므로 ARPU는 높을 것으로 예상. 연 ₩1억-₩5억 규모 시장 추정 (500-1000명 × ₩20만/년).

**Confidence Level:** Medium

**Key Assumptions:**
- 연간 이직률 10-20%
- 유료 도구 전환율 1-2%
- 국내 시장만 타겟 (언어/문화 장벽)

### 3. Competition: 5/10

**Evidence:**
- **직접 경쟁자:**
  - Poromy (https://poromy.ai.kr): K-STAR-K 프레임워크 기반 이력서 프롬프트 생성
  - 차이점: Poromy는 프롬프트만 제공, 사용자가 직접 입력해야 함

- **간접 경쟁자:**
  - 원티드/사람인/잡코리아: 이력서 관리 기능
  - LinkedIn: 경력 관리 + AI 이력서 작성 기능 추가 중
  - ChatGPT 직접 사용: 사용자가 직접 프롬프트 작성

- **대체재:**
  - 이력서 작성 대행 서비스 (₩5만-₩30만)
  - 커리어 코칭 (₩50만-₩200만)
  - 블라인드/오픈챗: 무료 면접 질문 공유

**Reasoning:**
중간 정도의 경쟁 강도. Poromy가 이미 유사 문제를 해결하고 있지만, Slack/Jira 자동 연동이라는 차별점이 있음. 하지만 ChatGPT의 보편화로 "AI 이력서 작성" 자체는 이제 차별점이 아님. 핵심은 "데이터 자동 수집" 부분.

**Confidence Level:** High

**Key Assumptions:**
- Poromy가 유료 모델로 전환 시 경쟁 심화
- LinkedIn이 한국 시장 공략 시 위협

### 4. Monetization: 6/10

**Evidence:**
- **유사 서비스 가격대:**
  - Poromy: 가격 정보 없음 (무료 또는 프리미엄 모델 추정)
  - 이력서 작성 대행: ₩50,000-₩300,000 (1회성)
  - 커리어 코칭: ₩500,000-₩2,000,000

- **제안 가격 모델:**
  - 무료: 기본 이력서 1개 생성
  - 프리미엄: ₩19,900/월 또는 ₩99,000/이직 시즌 (3개월)
    - Slack/Jira 연동 무제한
    - 채용공고 맞춤형 이력서 무제한
    - 면접 질문 데이터베이스 접근
    - 면접 연습 기록

**Reasoning:**
수익 모델은 명확하지만 **구조적 챌린지**가 있음:
- 이직은 1-2년에 한 번 하는 일 → 월 구독 유지 어려움
- "이직 성공 후 구독 취소" 예상 → Churn 매우 높을 것
- **개선안:** 월 구독보다는 "이직 패키지" (3개월 ₩99,000) 형태가 더 자연스러움

**Confidence Level:** Medium

**Key Assumptions:**
- 경력자들이 이력서 도구에 ₩10만-₩20만 지불 의향 있음
- 이직 성공 시 가치를 인정 (연봉 상승분 대비 저렴)

### 5. Differentiation: 7/10

**Evidence:**
- **핵심 차별점:**
  1. **Slack/Jira 자동 연동**: Poromy는 수동 입력, 이 서비스는 자동 추출
  2. **크라우드소싱 면접 질문**: 블라인드와 달리 구조화된 데이터
  3. **통합 플랫폼**: 이력서 작성 + 면접 준비 한 곳에서

- **Poromy와 비교:**
  - Poromy: 채용공고 → K-STAR-K 프롬프트 생성 → 사용자가 직접 작성
  - 이 서비스: Slack/Jira → 자동 업무 추출 → 채용공고 맞춤 변환 → 이력서 생성

**Reasoning:**
Slack/Jira 자동 연동은 명확한 차별점. 하지만:
- "AI 이력서 작성" 자체는 이제 흔함 (ChatGPT, Claude)
- 면접 질문 크라우드소싱은 **Cold Start 문제** (초기 데이터 없음)
- 핵심은 "자동화"인데, 이게 정확하게 작동해야 가치 인정받음

**Confidence Level:** Medium

**Key Assumptions:**
- Slack/Jira 데이터에서 의미 있는 업무 성과 추출 가능
- 사용자들이 회사 데이터 연동에 동의할 것

### 6. Time to Market: 5/10

**Evidence:**
- **개발 복잡도:**
  - Slack API 연동: 중간 (OAuth, 채널/메시지 파싱)
  - Jira API 연동: 중간 (이슈 검색, 필터링)
  - AI 이력서 생성: 쉬움 (OpenAI API)
  - 면접 질문 커뮤니티: 중간-어려움 (CRUD + 모더레이션)

- **MVP 범위:**
  - Phase 1: Slack 연동 + AI 이력서 생성 (2-3개월)
  - Phase 2: Jira 연동 추가 (1개월)
  - Phase 3: 면접 질문 커뮤니티 (2개월)

**Reasoning:**
사이드 프로젝트로 진행 시 **3-4개월** 예상. Slack/Jira API 연동이 생각보다 복잡:
- 각 회사마다 Slack/Jira 사용 패턴이 다름
- 데이터 파싱 정확도 확보 어려움
- 디버깅 시간 많이 소요

**Confidence Level:** Medium

**Key Assumptions:**
- Slack/Jira API 경험 없다고 가정 (학습 시간 포함)
- 주당 10-15시간 투자 가능

### 7. Solo/Small Team Fit: 6/10

**Evidence:**
- **1인 개발 가능 영역:**
  - MVP 개발 (3-4개월): ✅
  - 초기 고객 10-50명 서포트: ✅

- **1인 개발 어려운 영역:**
  - 커뮤니티 운영 (면접 질문 모더레이션)
  - 다양한 회사의 Slack/Jira 패턴 대응 (디버깅)
  - 고객 지원 (이력서는 민감한 데이터, 문의 많음)
  - 데이터 파싱 정확도 유지

**Reasoning:**
초기에는 1인 가능하지만, 성장 시 **운영 부담**이 커짐:
- Slack/Jira 연동은 회사마다 다르게 동작 → 디버깅 요청 많음
- 이력서 데이터는 민감 → 꼼꼼한 고객 지원 필요
- 면접 질문 커뮤니티는 스팸/저품질 콘텐츠 관리 필요

**개선안:** MVP에서 면접 질문 기능 제외, 이력서 생성에만 집중

**Confidence Level:** High

**Key Assumptions:**
- 초기 50-100명 수준까지는 1인 운영 가능
- 이후 CS 담당 필요

### 8. Skills Match: 8/10

**Evidence:**
- **필요 스킬:**
  - ✅ 풀스택 개발 (제공됨)
  - ✅ API 연동 (학습 가능)
  - ✅ AI/LLM 프롬프팅 (보편화됨)
  - ⚠️ 한국 이력서 문화 이해 (경력자라면 보유)
  - ⚠️ 커뮤니티 운영 경험 (선택적)

**Reasoning:**
풀스택 개발 가능하므로 기술적으로는 큰 문제 없음. Slack/Jira API 학습 곡선은 있지만 감당 가능. 오히려 **도메인 지식**(한국 이력서 문화, 채용 프로세스)이 더 중요할 수 있음.

**Confidence Level:** High

**Key Assumptions:**
- API 연동 경험 있거나 빠르게 학습 가능
- 본인도 경력자로서 이력서 작성 경험 있음

## Risk Analysis

### Critical Risks (Score <5)

**없음** - 모든 차원이 5점 이상

### Moderate Risks (Score 4-6)

1. **Time to Market (5/10)**: Slack/Jira 연동 복잡도
   - **Mitigation:** MVP에서 Slack만 지원, Jira는 Phase 2로 연기
   - **Mitigation:** 초기에는 특정 회사/패턴만 지원 (점진적 확대)

2. **Competition (5/10)**: Poromy 존재 + ChatGPT 보편화
   - **Mitigation:** "자동 데이터 수집"에 집중 (Poromy는 수동 입력)
   - **Mitigation:** 무료 사용자에게 가치를 빠르게 보여주기 (Slack 연동 후 1분 만에 초안 생성)

3. **Solo/Team Fit (6/10)**: 운영 부담
   - **Mitigation:** MVP 범위 축소 (면접 질문 제외)
   - **Mitigation:** 셀프서비스 위주 설계, CS 최소화

4. **Monetization (6/10)**: 월 구독 모델 Churn
   - **Mitigation:** "이직 패키지" 모델로 전환 (3개월 ₩99,000)
   - **Mitigation:** 연간 구독 할인 (₩199,000/년, 월 환산 ₩16,583)

5. **Market Size (6/10)**: 국내 시장 한정
   - **Mitigation:** 초기에는 국내 집중, PMF 달성 후 해외 확장 검토
   - **Mitigation:** B2B 모델 추가 (헤드헌팅 회사, 커리어 코칭 업체에 SaaS 제공)

### Strengths (Score >6)

1. **Problem Intensity (7/10)**: 실제 pain point
   - **How to leverage:** 타겟 마케팅 메시지 명확 ("이직 준비 1시간 → 10분으로")
   - **How to leverage:** 사용 후기/케이스 스터디 강조

2. **Differentiation (7/10)**: Slack/Jira 자동 연동
   - **How to leverage:** 이것을 핵심 메시지로 ("자동으로 정리되는 커리어")
   - **How to leverage:** 무료 체험에서 "자동화"를 강하게 경험시키기

3. **Skills Match (8/10)**: 풀스택 개발 가능
   - **How to leverage:** 빠른 프로토타이핑과 iteration
   - **How to leverage:** 사용자 피드백 즉각 반영

## Recommendation

**Overall Assessment:** ITERATE

**Reasoning:**

50/80 (6.25/10) 점수는 **중간 수준의 잠재력**을 의미합니다. 바로 GO 하기에는 몇 가지 개선이 필요하지만, 피벗할 정도로 나쁘지는 않습니다.

**핵심 문제점:**
1. **수익 모델 취약**: 월 구독은 이직 특성상 Churn이 높을 수밖에 없음
2. **범위 과다**: MVP에 3가지 기능 (Slack/Jira + 이력서 + 면접) 모두 포함 → 출시 지연
3. **경쟁 우위 불명확**: Poromy 대비 "자동화"가 핵심인데, 이게 정말 작동할지 검증 필요

**왜 ITERATE가 필요한가:**
- Time to Market을 단축하기 위해 MVP 범위 축소 필요
- 수익 모델을 "이직 패키지"로 재설계하여 Churn 리스크 감소
- Slack/Jira 자동 추출의 **기술적 타당성**을 먼저 검증 (PoC)

**Conditions for Proceeding:**
- [ ] MVP 범위 축소: Slack 연동 + 이력서 생성만 (면접 질문 제외)
- [ ] 수익 모델 변경: 월 구독 → 이직 패키지 (3개월)
- [ ] 기술 검증: Slack 데이터에서 의미있는 업무 성과 추출 가능한지 PoC (1주)
- [ ] 고객 검증: 10명의 경력자에게 아이디어 피칭, 5명 이상 긍정 반응

**Next Steps:**

### Immediate (이번 주)

1. **기술 PoC (Proof of Concept)**
   - 본인 Slack 워크스페이스 데이터로 테스트
   - "내가 뭘 했는지" 의미 있게 추출되는지 확인
   - 시간: 5-10시간

2. **고객 인터뷰 10명**
   - 타겟: 이직 준비 중이거나 최근 이직한 경력자
   - 질문:
     - "이력서 작성할 때 가장 힘든 점은?"
     - "Slack/Jira에서 자동으로 업무 성과 추출해준다면 쓸 의향 있나?"
     - "얼마까지 낼 수 있나?"
   - 시간: 5시간 (인터뷰 30분 × 10명)

3. **MVP 범위 재정의**
   - Phase 1: Slack 연동 + AI 이력서 생성 (면접 질문 제외)
   - Phase 2 (optional): Jira 연동 추가
   - Phase 3 (optional): 면접 질문 커뮤니티

### After Validation (PoC + 인터뷰 후)

**If 긍정적 (5명 이상 "쓰고 싶다" + PoC 성공):**
→ MVP 개발 시작 (2-3개월)

**If 부정적 (관심 없음 or PoC 실패):**
→ 피벗 옵션 고려:
- Option A: Poromy처럼 "프롬프트 생성"만 제공 (자동화 포기)
- Option B: 이력서 대신 "연봉 협상" 도구로 피벗 (연봉 데이터 크롤링 + 협상 전략)
- Option C: 다른 아이디어 탐색

## Success Metrics (If Proceeding)

### 3-month goals (MVP 완료 시점)
- **Technical:** Slack 연동 작동률 >90% (테스트 유저 기준)
- **Users:** 50명 베타 사용자 확보
- **Engagement:** 사용자 1명당 평균 3개 이상 이력서 생성
- **Revenue:** 5명 이상 유료 전환 (₩99,000 × 5 = ₩500,000)

### 6-month goals (Public Launch 후)
- **Users:** 200명 가입, 50명 유료 전환
- **Revenue:** ₩5,000,000 (50명 × ₩100,000)
- **Retention:** 이직 성공률 >30% (사용자 후기 수집)
- **NPS:** >40

### Decision points
- **1개월 시점 (PoC + 인터뷰 완료):**
