# Filestash Success Story

- **URL:** https://www.filestash.app

## The Story

Filestash는 Mickael Kerjean이 8년 이상 지속적으로 개발해온 오픈소스 파일 관리 플랫폼입니다. "What Dropbox should have been if it was based on SFTP, S3, FTP, WebDAV, Git"이라는 비전으로 시작했습니다.

핵심은 **확장 가능한 백엔드 인터페이스를 구축**한 것입니다. 이를 통해 다양한 스토리지 백엔드(SFTP, S3, FTP, WebDAV, Git, Minio, LDAP, CalDAV 등)를 지원하며, 기업들이 필요로 하는 커스터마이징을 유료로 제공합니다.

### 성장 경로

1. **오픈소스 공개:** GNU AGPL-3.0 라이선스로 커뮤니티 구축
2. **확장성 우선 설계:** 플러그인/커스터마이징 가능한 아키텍처
3. **엔터프라이즈 라이선싱:** 기업용 기능과 지원 유료화
4. **커스터마이징 서비스:** 기업별 맞춤 개발 수익화
5. **장기 지속:** 8년+ 꾸준한 개발로 신뢰 구축

## Reproducible Tactics

### 1. Open Core 비즈니스 모델
**구체적 행동:**
- 핵심 기능은 오픈소스로 공개 (GNU AGPL-3.0)
- 엔터프라이즈 기능은 유료 (화이트 라벨링, SLA 지원, 보안 패치 우선 제공)

**왜 효과적:**
- 오픈소스로 커뮤니티와 신뢰 구축
- 기업은 기꺼이 지원과 추가 기능에 지불
- 양쪽 시장(개인/기업) 모두 공략

### 2. 확장 가능한 아키텍처
**구체적 행동:**
- 다양한 스토리지 백엔드 지원 (SFTP, S3, FTP, WebDAV, Git 등)
- 플러그인 시스템으로 커스터마이징 가능
- "extensible backend interface"

**왜 효과적:**
- 기업별 니즈에 대응 가능
- 커스터마이징 수요 → 추가 수익원
- 경쟁자 대비 유연성 우위

### 3. 듀얼 라이선싱 전략
**구체적 행동:**
- 오픈소스: GNU AGPL-3.0 (기본 기능)
- 엔터프라이즈: 커스텀 라이선스 (추가 기능 + 지원)
- 기업이 오픈소스 repo에서 파생한 플러그인 개발 시 엔터프라이즈 계정 필요

**왜 효과적:**
- AGPL은 파생 작업 공개 강제 → 기업은 프라이빗 개발 위해 유료 라이선스 구매
- 합법적으로 오픈소스와 상업용 분리
- 양쪽 생태계 모두 건강하게 유지

### 4. 엔터프라이즈 서비스 번들
**구체적 행동:**
- 월 $500+ 구독에 포함:
  - SLA 지원
  - 화이트 라벨링
  - 보안 패치 우선 제공
  - 커스텀 Docker 이미지, deb/rpm 패키지
  - Off-the-shelf 플러그인
  - 설정 강화 컨설팅

**왜 효과적:**
- 기업은 번들로 구매 선호 (구매 결정 간소화)
- 높은 ARPU (월 $500+)
- 지속적인 관계 구축

### 5. 커스터마이징을 수익원으로
**구체적 행동:**
- 기업별 맞춤 플러그인 개발 유료 제공
- "companies requiring custom plugins derived from the open source repo can open enterprise accounts"
- 개발 중 개발자 지원 제공

**왜 효과적:**
- 일회성이 아닌 지속적 수익
- 기업의 특수한 니즈 충족
- 프로젝트별 높은 수익 (컨설팅 요금)

### 6. 다양한 배포 옵션
**구체적 행동:**
- Self-hosted (온프레미스)
- AWS Marketplace
- 서드파티 매니지드 호스팅 (Elest.io 등)

**왜 효과적:**
- 기업별 선호 배포 방식 대응
- 마켓플레이스로 추가 유입 채널
- 호스팅 파트너와 수익 공유

### 7. 장기적 신뢰 구축
**구체적 행동:**
- 8년+ 지속적인 개발
- 꾸준한 커밋과 릴리스
- 커뮤니티 피드백 반영

**왜 효과적:**
- 기업은 장기 지속성 있는 도구 선호
- 오픈소스 커뮤니티 신뢰 축적
- "이 프로젝트는 사라지지 않는다" 메시지

## Success Formula

```
오픈소스 코어 + 확장 가능한 아키텍처 + 엔터프라이즈 라이선싱 + 커스터마이징 서비스 = 풀타임 수입
```

## Lessons Learned

1. **오픈소스는 신뢰 구축 도구** - 무료 제공으로 커뮤니티와 신뢰 형성
2. **확장성이 수익성** - 커스터마이징 가능한 아키텍처가 추가 수익원
3. **AGPL은 상업화 무기** - 파생 작업 공개 강제로 기업 라이선스 유도
4. **엔터프라이즈는 기꺼이 지불** - 지원과 SLA에 월 $500+ 지불
5. **장기 게임이 승리** - 8년+ 꾸준한 개발이 신뢰와 수익으로 연결
6. **배포 옵션 다각화** - Self-hosted, 마켓플레이스, 매니지드 모두 제공

## Anti-Patterns to Avoid

- ❌ 너무 이른 폐쇄: 초기부터 유료 전용으로 하면 커뮤니티 구축 실패
- ❌ 확장 불가능한 설계: 커스터마이징 어려우면 엔터프라이즈 수익 불가능
- ❌ MIT/Apache 라이선스: AGPL 대신 관대한 라이선스 쓰면 상업화 어려움
- ❌ 단일 배포 옵션: 다양한 배포 방식 제공으로 시장 확대

## Related Patterns

- **Open Core Model** - 오픈소스 코어 + 상업용 확장
- **Dual Licensing** - 오픈소스/엔터프라이즈 듀얼 라이선스
- **Enterprise Add-On** - 기업용 추가 기능 유료화
- **Customization as Revenue** - 커스터마이징을 수익원으로
- **Community First** - 커뮤니티 구축 후 수익화
- **Long-term Commitment** - 장기적 개발로 신뢰 확보
