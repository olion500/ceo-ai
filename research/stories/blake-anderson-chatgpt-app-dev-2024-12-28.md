# Blake Anderson ChatGPT App Development Success Story

## The Story

### Problem Discovery

**Background:**
Blake Anderson, 23, studied finance and computer science in college but **had zero coding experience**. Despite his CS degree, he never learned practical programming.

**The Critical Moment:**
Facing a decision between traditional employment and entrepreneurship, Blake turned to ChatGPT and asked: **"How do I build mobile apps?"**

**Key Insight:**
Rather than spending months learning to code traditionally, Blake realized ChatGPT could write 95% of the code while he focused on:
- Product vision
- User experience
- Marketing strategy
- Business decisions

**Problem Identified:**
People struggle with specific social situations:
- Dating (conversation starters, flirting)
- Appearance anxiety (what can be improved)
- Fitness (tracking calories accurately)

### MVP Development

**App 1: Rizz GPT (later renamed Plug AI) - Dating Assistant**

**Timeline:** ~1 month from idea to launch

**Development Process:**
- **95% of code written by ChatGPT**
- Used iterative prompting: asked questions different ways when responses unclear
- Learned Swift UI and Xcode through ChatGPT guidance
- Started with design specs: "blue gradient background," "center the button"
- ChatGPT generated corresponding code
- Gradually developed coding comprehension through hands-on implementation

**Initial Features:**
- OCR (optical character recognition) for screenshot text extraction
- AI-powered response generation for dating conversations
- Context-aware suggestions (flirty vs. professional tone)
- Chat interface for practice conversations
- Freemium model with subscription unlock

**Tech Stack:**
- **Language:** Swift (iOS native)
- **IDE:** Xcode
- **AI:** ChatGPT (code generation + in-app AI responses)
- **OCR:** Image text extraction technology
- **Monetization:** In-app subscriptions via App Store

**Development Workflow:**
1. Blake describes feature in plain English
2. ChatGPT generates Swift code
3. Blake copies code into Xcode
4. Tests functionality
5. Iterates on bugs/improvements with ChatGPT
6. Repeat until feature complete

**Key Technical Innovation:**
- Prompt engineering to refine AI outputs for natural, context-appropriate responses
- Used ChatGPT both as coding assistant AND as the AI engine powering the app

**App 2: Umax - Appearance Analysis**

**Timeline:** ~1.5 months (faster than first app)

**Features:**
- Facial analysis via AI
- Actionable improvement suggestions (skincare, hairstyle, grooming)
- Positive, confidence-building feedback
- Focus on controllable factors (not genetics)
- $3.99 subscription for detailed analysis

**Technical Challenges:**
- Delivering sensitive feedback about appearance tactfully
- Balancing honesty with maintaining user confidence
- Prioritizing features users could actually control

**App 3: Cal AI - Calorie Tracking**

**Timeline:** Ongoing (in collaboration)

**Team Evolution:**
- Brought in two 17-year-old developers (Zach Gagri and Henry Langmack)
- Blake's role: Strategy, funding, mentorship
- Managed other apps while team built Cal AI

**Features:**
- AI-powered food photo recognition

- Personalized nutrition guidance
- Meal tracking automation

## All Tactics & Strategies Used

### Tactic 1: ChatGPT as Code Generator (95% AI-Written Code)
- **What They Did:** Used ChatGPT to write nearly all application code
- **How They Implemented:**
  1. Described feature in plain English: "I want a blue gradient background"
  2. ChatGPT generated Swift code
  3. Copied code into Xcode
  4. Tested functionality
  5. If bugs: described error to ChatGPT, got fix
  6. Iterated until feature complete
- **Context:** Zero prior coding experience, needed speed

  - First app built in 30 days (vs. 6+ months traditional)

  - Validated approach: non-developers can build apps
- **Time Investment:** 30 days full-time for first app, 45 days for second

- **What They Did:** Asked same question multiple ways when ChatGPT response unclear
- **How They Implemented:**

- **Context:** Learning curve for effective AI communication
- **Results Achieved:**
  - Gradually improved prompt quality
  - Faster development over time
  - Developed coding comprehension through iteration
- **Time Investment:** Initially 50% of time prompting, reduced to 20% by second app
