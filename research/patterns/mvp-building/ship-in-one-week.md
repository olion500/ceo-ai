# Ship in One Week (or Less)

**Category:** mvp-building
**Success Rate:** Very High (4/6 stories, all who used this succeeded)
**Time Investment:** 1-7 days
**Difficulty:** Hard (requires experience, focus, scope discipline)

## What Is This Pattern?

첫 버전을 1주일 이내에 만들고 출시하는 전략. 완벽함보다 속도를, 모든 기능보다 핵심 가치를 우선시합니다.

## How It Works

1. **Step 1: Identify Core Value** - 단 하나의 핵심 가치만 정의
2. **Step 2: Strip Everything Else** - 핵심 아닌 모든 기능 제거
3. **Step 3: Use Familiar Stack** - 새로운 기술 배우지 말고 아는 것 사용
4. **Step 4: Build Fast** - 1-7일 집중 개발
5. **Step 5: Launch with Bugs** - 완벽하지 않아도 출시
6. **Step 6: Iterate Based on Real Users** - 실제 사용자 피드백으로 개선

## Real Examples (Ranked by Speed-to-Market)

### 1. Tony Dinh (TypingMind) - 1 Day Build → $22K in Week 1
- **Context**: OpenAI API 발표 당일 (Mar 1, 2023) 아이디어 → 즉시 빌드
- **Execution**:
  - Mar 1: 아이디어 + 첫 버전 빌드 (1일)
  - Mar 2: 도메인 등록
  - Mar 6: 공개 런칭 (5일 다듬기)
  - **Total: 1일 빌드 + 5일 polish = 6일**
- **Results**:
  - Day 1: $1K
  - Day 2: $2K
  - Day 3: $4K
  - Week 1: $22K total
  - Year 1: $500K+
- **Key Insight**: "I built it in one day after ChatGPT API launch" - First-mover advantage worth more than polish

### 2. Marc Lou (ShipFast) - 7 Days Build → $36K Month 1
- **Context**: 16개 제품 경험으로 빠른 빌드 가능
- **Execution**:
  - Late Aug 2023: 아이디어
  - 1 week: 개발
  - Sept 1, 2023: 런칭
  - **Total: 7 days**
- **Results**:
  - 48 hours: $6K profit
  - Month 1: $36,400
  - "Earned more in 48 hours than all of 2022"
- **Key Insight**: "Practice shipping products more frequently to build 'muscle memory'" - Speed is a learnable skill

### 3. Bhanu Teja (SiteGPT) - 2-3 Weeks → $10K MRR Month 1
- **Context**: AI 붐 초기, 빠르게 움직여야 했음
- **Execution**:
  - Early Mar 2023: 아이디어
  - 2-3 weeks: 학습 + 빌드 (AI tech 새로 배움)
  - Late Mar 2023: 런칭 (bugs 있었지만)
  - **Total: 2-3 weeks**
- **Results**:
  - Day 1: 15K visitors, first customers
  - Week 1: 200+ paying customers
  - Month 1: $10K MRR
- **Key Insight**: "Get the working product to the hands of actual customers as fast as possible"

### 4. Luca Galante (Vampire Survivors) - 3-4 Months Initial → £40M
- **Context**: 처음 만든 상업 게임, 무직 상태
- **Execution**:
  - Dec 2020 - Mar 2021: 첫 버전 빌드 (3-4개월)
  - Mar 31, 2021: itch.io 무료 런칭
  - **Total: 3-4 months** (게임치고는 매우 빠름)
- **Results**:
  - itch.io: 100명 목표 실패
  - Steam 전환 후 대성공
  - £40M net worth by 2024
- **Key Insight**: "Bare bones" 버전도 핵심 게임플레이 있으면 충분

## Why This Works

**First-Mover Advantage:**
- 타이밍 = 모든 것
- TypingMind: OpenAI API 발표 당일 빌드 → Week 1에 $22K
- SiteGPT: AI 붐 초기 빠른 진입 → Month 1에 $10K MRR
- 늦으면 경쟁자가 이미 시장 점유

**Real User Feedback:**
- 추측 < 실제 사용 데이터
- 1주 빌드 + 실사용자 피드백 > 3개월 "완벽한" 빌드
- Marc Lou: "Get product to actual customers as fast as possible"

**Psychological Benefits:**
- 완벽주의 = 분석 마비
- 빠른 런칭 = 동기부여 유지
- 빠른 validation = 계속할지 포기할지 빨리 결정

**Lower Sunk Cost:**
- 1주 투자 실패 = OK, 다음으로
- 6개월 투자 실패 = 엄청난 타격
- Marc Lou 포트폴리오 접근: 빠르게 여러 개 테스트

## Prerequisites

- **Skills Required:**
  - **7년+ 개발 경험** (Tony Dinh 강조)
  - 익숙한 기술 스택
  - Scope 정의 능력 (무엇을 뺄지)
  - 집중력 (1주일 몰입)
- **Time Required:** 1-7 days full-time or 2-3 weeks part-time
- **Capital Required:** $0-500 (도메인, 호스팅)
- **Other Requirements:**
  - 버그 있어도 출시하는 용기
  - "Good enough" 만족 능력
  - 기존 인프라/스택 재사용 가능

## Common Mistakes

1. **Mistake: "새로운 기술 배우면서 빌드"**
   - **Why It Fails:** 학습 + 빌드 = 시간 폭발
   - **How to Avoid:** Tony: "Choose the stack that you are already familiar with"

2. **Mistake: "모든 기능 넣으려고 함"**
   - **Why It Fails:** Feature creep = 출시 지연
   - **How to Avoid:** 단 하나의 핵심 가치만, 나머지는 나중에

3. **Mistake: "완벽해질 때까지 기다림"**
   - **Why It Fails:** 완벽 = 영원히 안 옴
   - **How to Avoid:** Bhanu: "Launched with bugs intentionally"

4. **Mistake: "초보자가 1주일에 하려고 함"**
   - **Why It Fails:** 경험 없으면 불가능
   - **How to Avoid:** Tony: "I've been doing it for years" - 먼저 경험 쌓기

5. **Mistake: "혼자 모든 것 처음부터 만들기"**
   - **Why It Fails:** 시간 부족
   - **How to Avoid:** Bhanu: 기존 인프라 재사용 (Feather infrastructure)

6. **Mistake: "Perfect timing 기다리기"**
   - **Why It Fails:** Perfect timing = 놓침
   - **How to Avoid:** Tony: OpenAI API 발표 "그날 바로" 빌드 시작

## When to Use This Pattern

✅ **Good Fit When:**
- 이미 **7+ years 개발 경험** 있음 (Tony 강조)
- 익숙한 기술 스택 사용 가능
- Timing이 critical (AI 붐, API 발표 등)
- 핵심 가치 proposition이 명확함
- B2D (Business to Developer) 제품
- 기존 인프라 재사용 가능
- 실패해도 괜찮은 상황 (포트폴리오 접근)

❌ **Bad Fit When:**
- 개발 경험 1-2년 (경험 먼저 쌓기)
- 완전히 새로운 기술 배워야 함
- 규제가 심한 산업 (healthcare, finance - QA 필수)
- 하드웨어 관련 (물리적 제조 시간)
- 매우 복잡한 엔터프라이즈 솔루션
- 런칭 실패 = 엄청난 리스크 (첫 집 담보 등)

1. **개발 경험:** 7+ years >> 3-5 years > 1-2 years
2. **스택 익숙도:** 매일 쓰는 스택 >> 가끔 쓰는 >> 새로 배워야 하는
3. **타이밍:** API 발표 당일 >> 1주 후 >> 1달 후

5. **재사용:** 기존 인프라 있음 >> 처음부터 >> 새 기술 배우면서

## Technical Tips

**How to Build in 1 Week:**

1. **Day 1**: Core functionality only
   - Tony: ChatGPT interface + faster typing + search
   - Marc: Next.js boilerplate + Stripe + Auth + DB

2. **Day 2-3**: Essential polish
   - 기본 UI
   - 치명적 버그 수정
   - 결제 연동

3. **Day 4-5**: Landing page + deployment
   - 가치 proposition 명확히
   - 도메인 + 호스팅 설정

4. **Day 6-7**: Final polish + launch prep
   - 스크린샷
   - Product Hunt listing
   - Launch tweet 작성

**What to Skip:**
- ❌ Admin panel
- ❌ Analytics (나중에 추가)
- ❌ 모든 edge cases
- ❌ Perfect design (Good enough)
- ❌ Tests (controversial but true for 1-week build)
- ❌ Multiple pricing tiers (simple pricing)

**What to Include:**
- ✅ Core value delivery
- ✅ 결제 (Stripe 30분)
- ✅ Basic auth
- ✅ Landing page
- ✅ 배포
