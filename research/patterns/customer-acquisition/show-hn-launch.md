# Show HN Launch

## What Is This Pattern?

Show HN (Show Hacker News) 런칭은 Hacker News에 직접 제품을 게시하여 초기 사용자를 확보하는 전략입니다. 개발자, 스타트업, 기술 애호가들이 모인 커뮤니티에서 즉각적인 피드백과 초기 트랙션을 얻을 수 있습니다.

## How It Works

1. **제품 준비**: MVP가 사용 가능한 상태여야 함
2. **포스트 작성**: "Show HN: [Product Name] – [Clear Value Proposition]" 형식
3. **직접 참여**: 댓글에서 질문에 성실히 답변
4. **피드백 수용**: 비판적 의견도 개선 기회로 활용
5. **관계 구축**: 초기 사용자와 장기 관계 형성

## Real Examples (Ranked by Success)

### 1. Video Hub App (2018)
- **What happened**: November 2018 HN 런칭 후 8년간 지속적 수익 ($500/month)
- **Why it worked**: 개발자 대상 도구, 명확한 가치 제안, 진정성 있는 참여
- **Key insight**: 한 번의 런칭이 8년간 지속되는 트래픽으로 연결

### 2. DB Pro (2025)
- **What happened**: Show HN 런칭 후 첫 달 $1,000 MRR 달성
- **Why it worked**: 프라이버시 중심 포지셔닝, 커뮤니티 피드백 즉각 반영
- **Key insight**: 런칭 후 가격 모델 피벗 (구독 → 평생 구매 추가)

### 3. Aptakube (2022-2023)
- **What happened**: HN 런칭 후 €5,000/month 수익 달성
- **Why it worked**: K8s 커뮤니티의 실제 문제 해결, 기술적 차별화 (Tauri)
- **Key insight**: 사용자들이 먼저 결제 옵션 요청할 정도의 제품-시장 적합성

## Why This Works

### 즉각적 트래픽
- HN 프론트 페이지 = 수만 명의 방문자
- 개발자/얼리어답터 집중

### 질 높은 피드백
- 기술적으로 정교한 사용자들의 의견
- 제품 개선 방향 제시

### 장기적 SEO 효과
- HN 백링크 = 높은 도메인 권한
- Google 검색 결과에서 지속적 노출

### 신뢰 구축
- 투명성과 직접 소통으로 신뢰 형성
- 초기 사용자가 에반젤리스트로 전환

## Prerequisites

### 필수 요소
1. **작동하는 MVP**: 데모만으로는 부족, 실제 사용 가능해야 함
2. **명확한 가치 제안**: 3초 안에 이해 가능한 핵심 가치
3. **창업자의 시간**: 런칭 당일 최소 4-6시간 댓글 참여
4. **기술적 안정성**: 트래픽 급증에 대응 가능한 인프라

### 선호 조건
- 개발자 도구 또는 기술 제품
- 오픈소스 또는 투명한 접근 방식
- 독특하거나 혁신적인 해결책
- 가격이 합리적 (또는 무료 체험)

## Common Mistakes

### ❌ 스팸성 홍보
- 문제: 과도한 마케팅 언어, 진정성 부족
- 해결: 겸손하고 진솔한 어조 사용

### ❌ 댓글 무시
- 문제: 질문에 답하지 않으면 신뢰 상실
- 해결: 모든 질문에 최소 간단한 답변

### ❌ 비판 방어
- 문제: 부정적 피드백에 방어적 반응
- 해결: "좋은 지적입니다. 개선하겠습니다" 자세

### ❌ 준비 부족
- 문제: 서버 다운, 링크 깨짐, 회원가입 오류
- 해결: 사전 로드 테스트, 모니터링 준비

## When to Use This Pattern

### ✅ 이상적인 경우
- 개발자 도구, SaaS, 기술 제품
- 진정한 문제 해결에 집중
- 투명하고 진솔한 스토리가 있음
- 즉각적 피드백 받을 준비 됨

### ⚠️ 주의 필요
- 소비자 제품 (B2C)
- 복잡한 설명이 필요한 제품
- 완성도가 낮은 MVP

### ❌ 피해야 할 경우
- 스팸성 제품
- 기술 커뮤니티와 무관한 제품
- 피드백 받을 의지 없음

## Tactical Playbook

### 런칭 전 (1주일 전)
1. [ ] HN 계정 생성 (카르마 50+ 권장)
2. [ ] 서버/인프라 로드 테스트
3. [ ] 런칭 타이밍 선정 (미국 동부 시간 오전 8-10시 또는 12-2시)
4. [ ] FAQ 예상 질문 준비
5. [ ] 모니터링 도구 설정 (Sentry, New Relic 등)

### 런칭 당일
1. [ ] "Show HN: [Product] – [Clear One-liner]" 제목 작성
2. [ ] 짧은 소개 코멘트 추가 (스토리, 왜 만들었는지, 피드백 요청)
3. [ ] 모든 댓글에 1시간 이내 답변
4. [ ] 비판적 피드백도 감사 표현
5. [ ] 제품 개선 사항 실시간 반영 (가능한 경우)

### 런칭 후 (1주일)
1. [ ] 피드백 정리 및 우선순위 설정
2. [ ] 주요 개선사항 구현
3. [ ] 업데이트 코멘트로 진행 상황 공유
4. [ ] 초기 사용자에게 개별 감사 메시지
5. [ ] HN 스레드를 마케팅 자료로 활용 (소셜 프루프)

## Success Metrics

- **즉각 지표**: HN 프론트 페이지 도달, 100+ upvotes
- **트래픽**: 500-5,000+ 방문자 (제품에 따라)
- **전환**: 초기 사용자 10-100명
- **피드백**: 구체적이고 실행 가능한 개선 제안
- **장기**: 백링크로 인한 지속적 SEO 트래픽
