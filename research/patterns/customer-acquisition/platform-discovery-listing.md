# Platform Discovery Listing

## What Is This Pattern?

소프트웨어 발견 플랫폼 (AlternativeTo.net, Product Hunt, Capterra, G2 등)에 제품을 리스팅하여 사용자들이 자연스럽게 발견하도록 하는 전략입니다. 사람들이 적극적으로 도구를 찾는 플랫폼에서 자동 발견됩니다.

핵심은 **passive discovery**입니다. 한 번 설정하면 8년 이상 지속적으로 트래픽을 가져옵니다.

## How It Works

1. **플랫폼 식별**: 타겟 고객이 사용하는 발견 플랫폼 찾기
2. **프로필 생성**: 상세한 제품 설명, 스크린샷, 기능 리스트
3. **SEO 최적화**: 카테고리, 태그, 키워드 정확히 설정
4. **리뷰 축적**: 초기 사용자에게 리뷰 요청
5. **지속적 트래픽**: 검색하는 사용자들이 자동 발견

## Real Examples (Ranked by Success)

### 1. Video Hub App (8 years, $300-500/month sustained)
- **What happened**:
  - AlternativeTo.net 리스팅
  - 8년간 지속적으로 트래픽 유입
  - 100 sales/month sustained
- **Why it worked**:
  - "video organization tool" 검색 시 자동 노출
  - 사용자들이 대안 찾을 때 발견
  - 리뷰와 평점으로 신뢰 구축
- **Key metrics**:
  - 8 years continuous traffic
  - Primary discovery channel
  - $0 marketing spend
- **Key insight**: 한 번 리스팅 = 8년+ 트래픽 (패시브 인컴)

### 2. Aptakube (€5,000/month)
- **What happened**:
  - Kubernetes tool directories
  - Developer community listings
  - Word-of-mouth from discovery platforms
- **Why it worked**:
  - DevOps 엔지니어가 "k8s gui" 검색
  - 기존 도구 (Lens, k9s) 대안 찾는 사람들 타겟
- **Key insight**: Niche B2B tools = 발견 플랫폼 매우 효과적

### 3. DB Pro ($1K MRR first month)
- **What happened**:
  - Database tools directories
  - Chrome Web Store (플랫폼 discovery의 일종)
  - Developer tool aggregators
- **Why it worked**:
  - "database GUI" 검색 자동 노출
  - 카테고리 최적화
- **Key insight**: Chrome Web Store = 개발자 도구의 App Store

## Why This Works

### 높은 Intent
- 사용자가 적극적으로 도구 찾는 중
- "도구 필요" → "검색" → "발견" → "구매"
- 전환율 높음

### 지속적 트래픽
- 한 번 설정 = 계속 유입
- SEO처럼 compounding effect
- 8년+ 지속 가능

### 제로 비용
- 대부분 플랫폼 무료 리스팅
- 마케팅 비용 $0
- 시간 투자만 (2-4시간)

### 신뢰 구축
- 리뷰와 평점 = 사회적 증거
- 제3자 플랫폼 = 공신력
- 경쟁자와 직접 비교 가능

## Prerequisites

### 필수 요소
1. **명확한 카테고리**: "video organizer", "k8s GUI" 등
2. **실제 사용자**: 리뷰 받을 고객 필요
3. **품질 제품**: 낮은 평점 = 역효과
4. **스크린샷/데모**: 시각적 자료 필수

### 선호 조건
- B2B 또는 개발자 도구 (더 효과적)
- 기존 경쟁자 대안 포지셔닝
- 명확한 차별화 요소
- 무료 버전 또는 트라이얼

## Common Mistakes

### ❌ 불완전한 프로필
- 문제: 스크린샷 없음, 짧은 설명
- 해결: 상세한 설명, 고품질 이미지, 비디오

### ❌ 잘못된 카테고리
- 문제: "Productivity" 너무 broad
- 해결: 구체적 카테고리 ("Video Management")

### ❌ 리뷰 없음
- 문제: 평점/리뷰 없으면 신뢰 낮음
- 해결: 초기 10-20명 사용자에게 리뷰 요청

### ❌ 한 개 플랫폼만
- 문제: 트래픽 소스 단일화
- 해결: 10-20개 플랫폼 리스팅

## When to Use This Pattern

### ✅ 이상적인 경우
- **B2B SaaS**: 기업이 도구 찾을 때 사용
- **개발자 도구**: DevOps, 데이터, 인프라
- **생산성 도구**: 노트, 프로젝트 관리, 자동화
- **대안 제품**: 기존 인기 도구의 개선 버전

### ⚠️ 주의 필요
- **소비자 제품**: B2C는 효과 낮음 (앱스토어가 더 나음)
- **완전히 새로운 카테고리**: 아무도 검색 안 함
- **초기 MVP**: 리뷰 받을 사용자 없음

### ❌ 피해야 할 경우
- **낮은 품질**: 나쁜 리뷰 = 역효과
- **모바일 앱**: App Store가 더 적합
- **게임**: Steam, Epic이 더 적합

## Tactical Playbook

### 1단계: 플랫폼 리스트 작성 (1시간)
**Tier 1 (필수, 모든 제품)**:
1. [ ] Product Hunt - 런칭 플랫폼
2. [ ] AlternativeTo.net - 대안 검색
3. [ ] Capterra - B2B 도구
4. [ ] G2 - B2B 리뷰
5. [ ] TrustRadius - 엔터프라이즈

**Tier 2 (카테고리별)**:
- **Developer Tools**:
  - [ ] Stack Share
  - [ ] DevHunt
  - [ ] Awesome lists (GitHub)
- **SaaS Tools**:
  - [ ] SaaSHub
  - [ ] GetApp
  - [ ] Software Advice
- **Productivity**:
  - [ ] Notion Templates
  - [ ] Zapier App Directory
- **Chrome Extensions**:
  - [ ] Chrome Web Store (이미 있음)
  - [ ] Extension aggregators

**Tier 3 (니치별)**:
- Industry-specific directories
- Reddit sidebars (r/selfhosted 등)
- Awesome lists

### 2단계: 프로필 최적화 (2-3시간)
1. [ ] **제목**: 명확한 가치 제안 (30자 이내)
2. [ ] **설명**:
   - 첫 문장 = 핵심 가치
   - 3-5 bullet points = 주요 기능
   - 차별화 요소 강조
   - 200-500 단어
3. [ ] **스크린샷**:
   - 5-10장 고품질 이미지
   - 핵심 기능 강조
   - 1920x1080 해상도
4. [ ] **비디오**:
   - 1-2분 데모 (선택)
   - YouTube 업로드 후 링크
5. [ ] **카테고리/태그**:
   - 정확한 분류
   - 모든 관련 태그 추가
   - 경쟁자와 동일 카테고리

### 3단계: 리뷰 수집 (1-2주)
1. [ ] 초기 10-20명 사용자 식별
2. [ ] 개인 이메일로 리뷰 요청
3. [ ] 템플릿:
   ```
   Hi [Name],

   Thank you for using [Product]!

   Would you mind leaving a quick review on [Platform]?
   It would really help other users discover us.

   [Direct link to review page]

   Thanks!
   ```
4. [ ] 리뷰 답변 (감사 표현)
5. [ ] 4-5 star 평점 목표

### 4단계: SEO 최적화 (1시간)
1. [ ] 키워드 리서치:
   - 사용자가 검색하는 용어
   - 경쟁자 키워드
2. [ ] 제목/설명에 키워드 포함
3. [ ] 태그 최적화
4. [ ] 백링크 추가 (내 웹사이트 → 리스팅)

### 5단계: 모니터링 및 업데이트 (월 1회)
1. [ ] Google Analytics로 referral traffic 추적
2. [ ] 어떤 플랫폼이 가장 효과적인지 분석
3. [ ] 새로운 리뷰에 답변
4. [ ] 분기별 스크린샷 업데이트
5. [ ] 새로운 기능 추가 시 설명 업데이트

## Platform-Specific Tips

### AlternativeTo.net
- 대안으로 포지셔닝 ("Alternative to [Popular Tool]")
- 차별화 요소 명확히
- "Free" or "Open Source" 태그 = 더 많은 조회

### Product Hunt
- 런칭 전 "Coming Soon" 페이지
- 런칭 day: 화요일-목요일 오전 12:01 AM PST
- Hunter와 Maker 역할 명확히
- 첫 6시간이 critical

### Capterra / G2
- 기업 고객 타겟
- 케이스 스터디 추가
- 통합 파트너 명시
- ROI calculator (선택)

### Chrome Web Store
- SEO 제목: "Best [Category] Extension for [Use Case]"
- 5-star 리뷰 critical
- 업데이트 자주 (active development signal)
- Privacy policy 명확히

## Success Metrics

### 초기 (1-3개월)
- 10+ 플랫폼 리스팅 완료
- 평균 4+ star 평점
- 월 100+ profile views

### 중기 (3-6개월)
- Referral traffic > 500/month
- 플랫폼별 conversion tracking
- Top 3 카테고리 랭킹

### 장기 (6-12개월+)
- Referral traffic > 2,000/month
- 플랫폼이 top 3 traffic source
- 패시브 인컴 채널 확립

## Cost Structure

- **시간**: 초기 10-15 시간, 이후 월 1-2 시간
- **비용**: $0 (대부분 무료) - $50/month (premium listings 선택)
- **ROI**: 8년+ 지속 트래픽 = 매우 높음
