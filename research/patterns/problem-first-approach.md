# 패턴: Problem-First Approach (문제 우선 접근)

---

## 패턴 설명

모든 성공 사례에서 창업자가 **자신이 직접 겪은 문제** 또는 **관찰한 실제 문제**를 해결하는 것으로 시작했습니다. 트렌드를 쫓거나 "좋을 것 같은" 아이디어가 아닌, 검증된 pain point를 해결했습니다.

---

## 사례별 적용

### AI/SaaS Tools

**SiteGPT (Bhanu Teja)**
- **문제**: ChatGPT 웹사이트 인터페이스가 느리고, 검색 불가능, 생산성 부족
- **해결**: 더 나은 UI 레이어 제공
- **결과**: $10K MRR → $95K MRR (11개월)

**TypingMind (Tony Dinh)**
- **문제**: ChatGPT 인터페이스 느린 타이핑 속도, 대화 검색 불가
- **해결**: 개선된 인터페이스
- **결과**: 첫 7일 $22K, 현재 $83K MRR

**AnimeGenius (YiMeta)**
- **문제**: 일반 AI 아트 생성기는 애니메이션에 특화되지 않음
- **해결**: 애니메이션 전용 AI 생성기
- **결과**: $12K MRR (1년)

**Mockey.ai (Pranay Agarwal)**
- **문제**: 목업 생성이 비싸고 느리고 Photoshop 지식 필요
- **해결**: AI 기반 빠른 목업 생성기
- **결과**: $12K MRR (11개월)

### 인디 게임

**Vampire Survivors (Luca Galante)**
- **문제**: 인크리멘탈 게임이 시각적 개성과 전략 깊이 부족
- **해결**: 시각적으로 만족스럽고 전략적인 생존 게임
- **결과**: £40M 순자산, 120K+ 판매 첫 달

**Balatro (LocalThunk)**
- **문제**: 인크리멘탈 게임이 텍스트 기반이고 전략 깊이 부족
- **해결**: 포커 테마의 로그라이크 덱빌딩
- **결과**: 5M+ 판매, $20M+ 수익

**Gnorp Apologue (Myco)**
- **문제**: 인크리멘탈 게임이 시각적 개성 부족
- **해결**: 귀여운 Gnorp 캐릭터와 전략적 깊이
- **결과**: 120K+ 판매 첫 달 (~$600K)

### 이커머스 (Etsy)

**PokeMinh (Minh + Thao)**
- **문제**: Pokemon 카드 디스플레이 케이스가 비싸고 대량 구매 필요
- **해결**: 개별 판매, 커스텀 아트 삽입
- **결과**: 22,808 판매 (3년), $600K+ 수익

**FabledFraming (Harry)**
- **문제**: 트레이딩 카드 프레임이 비싸거나 커스터마이징 불가
- **해결**: 주문 제작 커스텀 프레임
- **결과**: 7,374 판매 (6년), $250K+ 수익

**RedDragonDesignsUK (Alunjohn Jones)**
- **문제**: 전통적인 카드 디스플레이가 평면적이고 예술성 부족
- **해결**: 3D 섀도우박스 + 커스텀 배경
- **결과**: 1,900 판매 (8년), $150K+ 수익

## 핵심 요소

### 1. 자신이 첫 고객

**특징:**
- 창업자가 직접 제품 사용
- 자신의 워크플로우에서 pain point 발견
- "Would I pay for this?" 자문

**예시:**
- Bhanu Teja (SiteGPT): Feather 개발하며 ChatGPT 불편함 경험
- Tony Dinh (TypingMind): ChatGPT 매일 사용하며 한계 발견
- Minh + Thao (PokeMinh): Pokemon 카드 수집하며 디스플레이 문제 경험

### 2. 관찰된 문제

**특징:**
- 고객이나 동료의 문제 관찰
- 클라이언트 작업 중 발견
- 커뮤니티에서 반복되는 불만

**예시:**
- Pranay Agarwal (Mockey.ai): Blinkstore.in에서 클라이언트 니즈 관찰
- YiMeta (AnimeGenius): 애니메이션 팬들이 기술 부족한 것 관찰

### 3. "Painkiller, Not Vitamin"

**검증 기준:**
- **진통제 (Painkiller)**: 당장 불편하고 해결 필요 ✅
  - 예: ChatGPT 느린 타이핑, 목업 생성 20-30분 소요
- **비타민 (Vitamin)**: 있으면 좋지만 급하지 않음 ❌
  - 예: "AI 뉴스 큐레이션", "소셜 미디어 분석 대시보드"

**차이:**
- 진통제: "지금 당장 필요해!"
- 비타민: "나중에 살게요..."

## 재현 가능한 프로세스

### Step 1: 문제 발견 (1-2주)

**방법:**
1. **자신의 워크플로우 감사**
   - 매일 하는 반복 작업 기록
   - 20분+ 걸리는 작업 찾기
   - "이거 더 빠르면 좋을 텐데" 순간 포착

2. **불만 수집**
   - 동료/친구에게 불편한 점 질문
   - Reddit, Twitter에서 "I hate", "frustrated" 검색
   - 커뮤니티에서 반복되는 문제 관찰

3. **기존 솔루션 평가**
   - 현재 사용 중인 불완전한 해결책?
   - Photoshop 같은 복잡한 도구 사용?
   - 매번 수동으로 처리?

### Step 2: 문제 검증 (3-7일)

**질문:**
- [ ] 이 문제를 나 말고도 10명 이상 겪는가?
- [ ] 현재 불완전한 방식으로라도 해결하려 하는가?
- [ ] 해결책에 돈을 낼 의향이 있는가?
- [ ] 문제가 자주 발생하는가? (매일/매주)

**검증 방법:**
- 5명에게 설명했을 때 "오, 나도 그거 필요해!" 반응
- Reddit/Twitter에 포스트했을 때 upvote/좋아요
- 비슷한 문제 해결하는 기존 제품 존재 (수요 증명)

### Step 3: 경쟁 분석 (1주)

**기존 솔루션이 있을 때:**
- ✅ **좋은 신호**: 시장 검증됨, 사람들이 돈 낼 의향 있음
- ✅ **차별화 기회**: 10배 빠르게/쉽게/저렴하게 만들 수 있는가?

**기존 솔루션이 없을 때:**
- ⚠️ **질문**: 왜 아무도 안 만들었나?
- 기술적으로 불가능? → AI/새 기술로 가능해졌나?
- 시장이 없나? → 니치하지만 충분한가?

## 안티패턴 (피해야 할 것)

### ❌ 트렌드만 쫓기

**사례:**
- Edmund Yong (Easy Folders): "트렌드를 맹목적으로 쫓지 말라"
- Mockey.ai: AI 뉴스 콘텐츠 실패 (Google 패널티)

**문제:**
- 실제 pain point 없음
- 차별화 어려움
- 시장 과포화

### ❌ "빌드하면 올 거야" (Build It and They Will Come)

**사례:**
- Vampire Survivors: itch.io에서 100명 목표도 실패
  - 하지만 Steam으로 플랫폼 변경 → 성공

**문제:**
- 유통 채널 없음
- 타겟 고객 불명확
- 시장 검증 없음

### ❌ 너무 넓은 타겟

**사례:**
- SEOmatic: 처음에 "모든 사람"을 타겟 → churn 높음
  - 재포지셔닝 → 디지털 에이전시 전용 → 성장

**문제:**
- 마케팅 어려움
- 제품 메시지 약함
- 차별화 불가능

## 성공 지표

### 좋은 문제의 신호

**✅ 즉각적 관심:**
- 설명 후 "When can I use it?" 질문
- "Please notify me when it launches" 이메일
- "I'd pay $X for this" 반응

**✅ 기존 불완전한 솔루션:**
- 사람들이 Photoshop으로 20분 작업
- 수동으로 매번 복붙
- 여러 도구 조합해서 사용

**✅ 반복 발생:**
- 매일 또는 매주 문제 경험
- 비즈니스 크리티컬한 작업
- 시간/돈 절약 가치 명확

### 나쁜 문제의 신호

**❌ 미온적 반응:**
- "That's interesting..." (관심 없음)
- "Maybe someday..." (급하지 않음)
- "Let me think about it..." (필요 없음)

**❌ 해결 안 하려 함:**
- 현재 불편해도 그냥 참음
- 무료 대안으로 충분
- 돈 낼 의향 없음

## 패턴 적용 체크리스트

**프로젝트 시작 전:**
- [ ] 나 또는 아는 사람이 이 문제로 매주 불편함
- [ ] 현재 불완전한 방식으로 해결 시도 중
- [ ] 시간/돈을 절약할 수 있는 명확한 가치
- [ ] 비슷한 문제 겪는 사람 10명+ 찾음
- [ ] 경쟁 제품 있거나 (시장 검증) 없어도 이유 명확

**MVP 개발 중:**
- [ ] 나 자신이 매일 사용할 수 있는 수준
- [ ] 핵심 pain point만 해결 (완벽 추구 금지)
- [ ] 5명 베타 테스터가 "정말 필요해" 반응

**론칭 후:**
- [ ] 고객들이 구체적인 pain point 언급
- [ ] "This saves me X hours/week" 피드백
- [ ] 낮은 refund rate (진짜 문제 해결 증명)

## 결론

**핵심 교훈:**
> "Make something people want" - Paul Graham, Y Combinator

모든 13개 사례에서 공통점:
1. 실제 문제 존재
2. 창업자가 직접 경험하거나 관찰
3. 불완전한 기존 해결책
4. 명확한 가치 제안

**행동 단계:**
1. 오늘: 워크플로우에서 불편한 점 3개 적기
2. 이번 주: 5명에게 같은 문제 겪는지 확인
3. 다음 주: 가장 심각한 문제 하나 선택, MVP 계획
