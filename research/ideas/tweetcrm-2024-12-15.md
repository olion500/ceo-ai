## Problem & Solution

### The Problem
**Who:** Building in Public founders (#buildinpublic creators)
**What:** Twitter DMs for customer support = unmanageable, no search, no organization
**Current Pain:**
- DMs = no search, no tags, no history
- Lose track of customer conversations
- Miss follow-ups
- Can't categorize inquiries

**Why It Hurts:**
- Important customer messages get lost
- No way to track customer history
- Unprofessional (slow responses)

### The Solution
Twitter API → Fetch DMs → Kanban board interface → Add tags, notes, status tracking

**Core Value Proposition:**
Help Building in Public founders manage customer DMs professionally using CRM-style organization for $19/month.

---

## Product Specification

### MVP Scope (7-10 days)

**Core Features:**
1. Twitter API OAuth
2. DM sync
3. Kanban board (New/In Progress/Done)
4. Tags and notes per conversation

**Excluded:**
- ❌ Automated responses
- ❌ Team features
- ❌ Analytics

### Technical Requirements

**Stack:**
- Frontend: Next.js + React
- Backend: Next.js API + Twitter API
- Database: Supabase (conversation data)

**Required Skills:**
- React: Advanced
- Twitter API: Beginner (need 2-3 days)
- Kanban UI: Intermediate

**Skill Gaps:**
- Twitter API OAuth + DMs: 2-3 days
